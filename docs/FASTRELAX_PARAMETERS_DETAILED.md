# FastRelax 参数详细说明

## 📚 参数意义与作用机制

---

## 1. `constrain_relax_to_start_coords`（约束到起始坐标）

### 含义

**是否限制 Relax 后的结构不能偏离原始结构太远**

### 工作原理

```python
fr.constrain_relax_to_start_coords(True)  # 启用约束
```

**启用时（True）**：
- PyRosetta 会在能量函数中添加**约束项**
- 约束项惩罚结构偏离原始坐标
- 公式：`总能量 = 物理能量 + λ × 约束能量`
- 其中 `λ` 是约束权重，通常逐渐减小（从强约束到弱约束）

**禁用时（False）**：
- 只优化物理能量，不限制结构变化
- 允许更大的结构变化

### 实际影响

| 设置 | 结构变化范围 | 优化程度 | 保持原始结构 | 适用场景 |
|------|------------|---------|-------------|---------|
| **True** | 小（通常 < 1-2 Å RMSD） | 中等 | ✅ 保持 | AF3预测结构（保持折叠） |
| **False** | 大（可能 > 2-5 Å RMSD） | 更彻底 | ❌ 可能改变 | 对接结构（允许更大优化） |

### 示例

**场景1：AF3 预测的抗体-抗原复合物**
```
原始结构：AF3 预测（质量较好）
约束=True：保持 AF3 的折叠，只修正局部不合理性
约束=False：可能改变整体构象，偏离 AF3 预测
```

**场景2：对接得到的复合物**
```
原始结构：对接结果（可能有冲突）
约束=True：在对接结构附近优化
约束=False：允许更大的结构重排
```

### 建议

- **AF3 预测结构**：建议使用 `True`（保持预测的折叠）
- **对接结构**：可以使用 `False`（允许更大优化）
- **大规模筛选**：使用 `False`（更快，且筛选阶段不需要严格保持结构）

---

## 2. `min_type`（最小化类型）

### 含义

**用于能量最小化的算法类型**

### 可选值

| 算法 | 全称 | 特点 | 速度 | 内存 |
|------|------|------|------|------|
| **默认** | 梯度下降 | 简单稳定 | 中等 | 低 |
| **`"lbfgs_armijo_nonmonotone"`** | LBFGS + Armijo线搜索 | 高级，收敛快 | 快 | 中等 |
| **`"dfpmin_armijo_nonmonotone"`** | DFP + Armijo | 类似LBFGS | 快 | 中等 |
| **`"lbfgs_armijo"`** | LBFGS + Armijo（单调） | 更保守 | 中等 | 中等 |

### LBFGS 算法详解

**LBFGS** = Limited-memory Broyden–Fletcher–Goldfarb–Shanno

**工作原理**：
1. **拟牛顿法**：近似 Hessian 矩阵（二阶导数）
2. **有限内存**：只保存最近的梯度信息（不存储完整矩阵）
3. **Armijo 线搜索**：自动调整步长，确保能量下降
4. **非单调**：允许能量暂时上升（跳出局部最小值）

**优势**：
- ✅ **收敛更快**：通常需要更少的迭代次数
- ✅ **更智能**：利用历史梯度信息
- ✅ **跳出局部最小值**：非单调版本可以跳出局部最优

**劣势**：
- ⚠️ **内存稍高**：需要存储梯度历史
- ⚠️ **可能不稳定**：某些情况下可能不收敛

### 实际影响

**使用默认算法**：
```python
fr.min_type(None)  # 或 不设置
```
- 简单梯度下降
- 稳定可靠
- 适合大规模批量处理

**使用 LBFGS**：
```python
fr.min_type("lbfgs_armijo_nonmonotone")
```
- 收敛更快（可能减少 20-50% 迭代次数）
- 但内存占用稍高
- 适合精细优化

### 性能对比（示例）

对于相同的结构，达到相同能量：

| 算法 | 迭代次数 | 时间 | 内存 |
|------|---------|------|------|
| 默认 | 100次 | 60秒 | 100MB |
| LBFGS | 60次 | 40秒 | 120MB |

**注意**：实际效果取决于结构复杂度

### 建议

- **大规模筛选**：使用默认算法（更稳定）
- **精细优化**：使用 LBFGS（更快收敛）
- **内存受限**：使用默认算法

## 3. `set_jump(False)`（禁止整体链移动）

### 含义

**是否允许整个链（或复合物）的相对位置改变**

### 工作原理

**Jump** = 链之间的刚体变换（旋转+平移）

```python
mm.set_jump(False)  # 禁止整体移动
```

**False（禁止）**：
- 保持链之间的相对位置不变
- 只允许链内部的局部优化（骨架、侧链）
- 适用于：保持复合物的相对位置

**True（允许）**：
- 允许链之间相对移动
- 可能改变复合物的整体构象
- 适用于：对接优化

### 实际影响

**示例：抗体-抗原复合物**

```
set_jump(False)：
  - 保持抗体和抗原的相对位置
  - 只优化界面附近的局部结构
  - 保持整体结合模式

set_jump(True)：
  - 允许抗体和抗原相对移动
  - 可能改变结合角度
  - 可能找到更好的结合模式（但也可能偏离原始）
```

### 建议

- **AF3 预测的复合物**：使用 `False`（保持预测的结合模式）
- **对接结构**：可以使用 `True`（允许优化结合模式）
- **界面分析**：通常使用 `False`（保持相对位置）

## 4. 对齐步骤（AlignChainMover）

### 含义

**Relax 后将结构对齐回原始结构**

### 工作原理

```python
from pyrosetta.rosetta.protocols.simple_moves import AlignChainMover

align = AlignChainMover()
align.source_chain(0)      # 源链（原始结构）
align.target_chain(0)       # 目标链（Relax后的结构）
align.pose(start_pose)      # 设置原始结构
align.apply(pose)           # 对齐 Relax 后的结构
```

**作用**：
1. 计算原始结构和 Relax 后结构的最佳对齐（最小化 RMSD）
2. 旋转和平移 Relax 后的结构，使其与原始结构对齐
3. **不改变内部结构**，只改变整体位置和方向

### 为什么需要对齐？

**问题**：
- Relax 可能改变结构的**整体位置和方向**
- 但**内部结构**（相对坐标）已经优化

**对齐的作用**：
- 保持结构的**空间位置**一致
- 便于后续分析（如轨迹分析、结构比较）
- 保持 B factors 等元数据的一致性

### 实际影响

**不使用对齐**：
```
原始结构：位置 A，方向 X
Relax后：  位置 B，方向 Y（内部已优化）
→ 位置和方向都改变了
```

**使用对齐**：
```
原始结构：位置 A，方向 X
Relax后：  位置 B，方向 Y（内部已优化）
对齐后：   位置 A，方向 X（内部仍优化）
→ 只改变内部结构，位置和方向保持一致
```

### 建议

- **筛选阶段**：通常不需要对齐（只关心能量，不关心位置）
- **结构输出**：如果需要保存 Relax 后的结构，建议对齐
- **轨迹分析**：必须对齐（保持时间序列的一致性）

## 5. `max_iter`（最大迭代次数）

### 含义

**FastRelax 的最大迭代轮数**

### 工作原理

FastRelax 是**迭代优化**过程：

```
第1轮：强约束优化（λ大，约束强）
第2轮：中等约束优化（λ中等）
第3轮：弱约束优化（λ小）
...
第N轮：无约束优化（λ=0，纯物理能量）
```

**每轮迭代**：
1. 设置约束权重 λ
2. 运行能量最小化
3. 检查收敛
4. 如果未收敛，进入下一轮

**`max_iter` 的作用**：
- 限制最大迭代轮数
- 通常 1-5 轮就足够
- 更多轮数：更彻底，但收益递减

### 实际影响

| max_iter | 优化程度 | 时间 | 能量改善 | 适用场景 |
|---------|---------|------|---------|---------|
| **1** | 快速优化 | 30-60秒 | 60-80% | 大规模筛选 ✅ |
| **2** | 中等优化 | 60-120秒 | 80-90% | 中等规模筛选 |
| **3** | 较好优化 | 90-180秒 | 90-95% | 小规模精细筛选 |
| **5** | 很好优化 | 150-300秒 | 95-98% | 单个结构优化 |
| **200** | 非常彻底 | 3-10分钟 | 99%+ | 精细优化（Germinal） |

**能量改善**：相对于完全不 Relax 的能量降低百分比

### 建议

- **大规模筛选（313个结构）**：`max_iter=1` ✅
- **中等规模（50-100个）**：`max_iter=2`
- **精细优化（<20个）**：`max_iter=3-5`

## 📊 参数组合建议

### 组合1：快速筛选（当前默认，推荐）

```python
fr.max_iter(1)
# 不设置 constrain_relax_to_start_coords（默认False）
# 不设置 min_type（使用默认）
# 不设置 set_jump（默认True，但通常不需要）
# 不对齐
```

**适用**：313个结构的大规模筛选  
**时间**：30-60秒/结构  
**特点**：快速，结果通常足够准确

---

### 组合2：平衡模式（推荐用于最终筛选）

```python
fr.max_iter(2)
fr.constrain_relax_to_start_coords(True)          # 保持结构
fr.min_type("lbfgs_armijo_nonmonotone")          # 快速收敛
mm.set_jump(False)                                # 保持相对位置
# 可选：对齐步骤
```

**适用**：Top 50-100 个候选的精细评估  
**时间**：60-120秒/结构  
**特点**：更彻底，保持结构

---

### 组合3：精细优化（Germinal 风格）

```python
fr.max_iter(200)
fr.constrain_relax_to_start_coords(True)
fr.min_type("lbfgs_armijo_nonmonotone")
mm.set_jump(False)
# 对齐步骤
align.apply(pose)
```

**适用**：单个结构的精细优化  
**时间**：3-10分钟/结构  
**特点**：非常彻底，保持结构和对齐

## 🎯 总结

| 参数 | 作用 | 启用/设置的影响 | 禁用/默认的影响 |
|------|------|----------------|----------------|
| **constrain_coords** | 限制结构变化 | 保持原始结构，更保守 | 允许更大变化，更彻底 |
| **min_type** | 最小化算法 | LBFGS：更快收敛 | 默认：更稳定 |
| **set_jump** | 整体移动 | False：保持相对位置 | True：允许相对移动 |
| **对齐** | 空间位置 | 保持位置一致 | 位置可能改变 |
| **max_iter** | 迭代次数 | 更多：更彻底但更慢 | 更少：更快但可能不够 |
