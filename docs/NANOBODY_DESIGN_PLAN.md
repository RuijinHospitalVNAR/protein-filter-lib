# 纳米抗体（VHH）设计计划书
> 基于 `protein_filter_lib` 软件的设计思路

---

## 一、总体设计思想

`protein_filter_lib` 的核心定位是：**用计算筛选替代湿实验试错**，从大量候选结构中逐级收敛到少数最佳设计。

针对纳米抗体的特殊性——CDR3长度短（3-25 aa）、依赖H3 loop 插入抗原表位口袋、分子量仅12-15 kDa——筛选策略必须兼顾**结构合理性**（折叠稳定）和**结合特异性**（界面互补）。

### 设计哲学：用计算指标做三层漏斗

```
AF3 预测（数百~数万候选）
    ↓ Stage 1: 置信度筛选（粗筛）
PyRosetta 静态打分（数十~数百候选）
    ↓ Stage 2: 物理化学评估（精选）
AMBER MD + MM/GBSA 动态打分（少数候选）
    ↓ 
最终选出 3-5 个候选做实验验证
```

---

## 二、两模式设计策略

### 模式 A：De novo 从头设计

**应用场景**：全新的靶点，没有已知的纳米抗体，需要通过噬菌体库或AI生成大量序列做 AF3 预测。

**流程**：

```
输入：靶点序列 + 随机CDR3库 / AntiBMPNN / SaProt 生成序列
  → AF3 预测（批量预测，数万结构）
  → Part 1: 三级筛选+聚类
  → Part 2: PyRosetta 静态分析
  → Part 3: MD 动态模拟
  → 实验验证
```

**关键决策点**：
- AF3 预测后先用各种内置置信度指标过滤掉明显不靠谱的结构（如 pLDDT < 0.7 的结构极可能不稳定）
- 聚类步骤可以找到不同的结合模式，避免"所有候选都是同一个binding pose"
- 对于纳米抗体，特别关注 CDR3 loop 的构象多样性

### 模式 B：亲和力成熟

**应用场景**：已有起始结构（可能来自晶体结构或之前的 AF3 预测），需要优化关键残基以提高亲和力。

**流程**：

```
输入：模板结构 + 突变设计方案（单点突变或 combinatory library）
  → AF3 预测（少量突变体，几十~几百个）
  → 跳过 Part 1（结构少，无需大量筛选）
  → Part 2: PyRosetta 静态分析（快速筛选突变体）
  → Part 3: MD + MM/GBSA（验证最终候选）
  → 实验验证
```

## 三、Part 1：AF3 置信度筛选与聚类（粗筛）

### 3.1 Stage 1 — 置信度筛选

| 指标 | 阈值 | 对纳米抗体的意义 |
|------|------|-----------------|
| **pLDDT** | ≥ 0.7 | 结构域整体可信度，纳米抗体结合后若CDR3区域pLDDT低说明可能未正确折叠 |
| **iPTM** | ≥ 0.6 | 界面预测TM-score，直接衡量结合的可靠性 |
| **ranking_confidence** | ≥ 0.7 | 0.8×ipTM + 0.2×pTM，AF3自己的综合排序指标 |
| **pDockQ** | ≥ 0.2 | 对接质量分数，>0.23 即"可能对接成功" |
| **clashes** | < 5 | 空间碰撞数，纳米抗体CDR3插入口袋时容易产生位阻 |
| **ipSAE** | ≥ 0.6 | 界面SAE分数，反映界面残基的构象可信度 |

**选中原因**：这些指标各覆盖结构的不同方面且彼此互补，联合使用不会因为某个指标的个别缺陷而错误保留或丢弃。pLDDT衡量折叠置信度，iPTM和pDockQ衡量对接成功概率，clashes确保结构物理合理性。

**纳米抗体特殊考虑**：纳米抗体的 CDR3 往往较长且灵活，AF3 预测时 H3 loop 的 pLDDT 可能偏低。如果已知抗原有特殊的结合沟槽结构，可以适当降低 pLDDT 阈值到 0.6，保留更多候选做后续精细分析。

### 3.2 Stage 2 — Foldseek 粗聚类

**原理**：通过 Foldseek 的 3Di 序列比对，对 AF3 预测的整体结构做无偏聚类。

**目的**：将候选结构按整体折叠模式分组，确保不会把全部候选都压在一个结合模式上，同时避免计算全量距离矩阵（O(N^2)的内存开销）。

**参数**：
- 灵敏度（sensitivity）：7.5（推荐值，平衡速度和精度）
- 最小簇大小：5（过滤掉没有代表性的奇异结构）
- 典型结果：18K 结构 → 25-50 个粗簇

**对纳米抗体的意义**：纳米抗体的 CDR3 长度差异很大（3-25 aa），不同长度的 CDR3 可能导致不同的结合模式。Foldseek 粗聚类能天然把这些差别较大的结构分开，让后续分析更有针对性。

### 3.3 Stage 3 — 簇内 H-A 接触精细聚类

**原理**：在每个粗簇内部，基于抗原-纳米抗体界面的接触残基对做聚类，找出同一折叠模式下的不同结合模式。

**方法**：支持 kmeans / HDBSCAN / DBSCAN 三种算法。

**关键参数**：
- contact_cutoff: 5.0 Å（原子间接触的定义距离）
- interface_cutoff: 8.0 Å（界面区域的识别范围）

**输出**：每个精细簇的代表结构，供下游 Part 2 分析。

**选中原因**：Stage 2 按整体结构相似性分组，但同一个蛋白折叠模式下可能有两种不同的结合姿势（比如 CDR3 插入的深度或方向不同）。精细聚类专门针对界面接触模式，能把这些隐藏的多样性找出来。

### 3.4 Stage 1 可选：界面接触聚类优先

在计算 Stage 1 指标之前，还可以增加一个**可选的聚类前置步骤**（`clustering_config`）——先基于界面的 H-A 接触做快速聚类，选择每个簇的代表性结构再计算指标。对这个设计的考虑是：

- 当总候选结构数量很大（几万级）时，先用界面接触空间将结构分组，可以跳过大量重复的、相似的结构，只对代表性结构进行完整的指标计算
- 这在计算资源和时间有限时非常实用
- trade-off：可能会丢失一些罕见但高置信度的结合模式

## 四、Part 2：PyRosetta 静态物理分析（精选）

### 4.1 核心指标

| 指标 | 含义 | 对纳米抗体的意义 |
|------|------|-----------------|
| **interface_dG** | 界面结合自由能（Rosetta能量函数） | 越负越好，判断突变体是否比WT结合更强 |
| **SAP_score** | 表面疏水性 | 纳米抗体的结构域外表面疏水性高可能带来聚集风险 |
| **delta_unsatHbond** | 界面氢键净变化 | 纳米抗体与抗原之间的极性相互作用 |
| **packstat** | 包装质量 | CDR3 loop 自身以及和 FR 区域的包装致密性 |

### 4.2 结构 Relax

**流程**：对 Part 1 选出的结构做 PyRosetta 的 fast_relax，然后计算界面指标。

**目的**：Relax 是一种快速的局部结构优化方法，可以去除 AF3 预测中可能存在的小的几何不合理性，使界面能量更可靠。另一方面它不会改变整体骨架构象。

**选中原因**：AF3 产生的结构虽然整体质量高，但局部残基 torsion angle 仍可能有物理不合理的地方。在进入昂贵的 MD 以前先用半小时做一轮 relax，可以有效过滤掉那些连 relax 都会导致结构显著退化的候选。

**Relax 筛选**：relax 后的结构若发生骨架较大变化（RMSD > 2 Å），说明该预测结构本身不稳定，应被淘汰。

### 4.3 筛选策略

1. 按 interface_dG 排序，选 Top-N（典型值 10-30）
2. 排除 SAP_score > 阈值的候选（避免聚集问题）
3. 排除 relax 后 RMSD 过大的候选（结构不稳定）
4. 保留 CDR3 序列有代表性多样性的候选

**阈值选择**：经验值来自文献（Germinal 相似筛选），但实际项目通常会先跑一个批次看分布再调整。

## 五、Part 3：MD + MM/GBSA 动态模拟（终选）

### 5.1 AMBER MD 模拟流程

```
输入结构（relax 后 PDB）
  ↓ tleap 构建体系
  ↓ 能量最小化（steepest descent + conjugate gradient）
  ↓ NVT 加热（0 → 310K）
  ↓ NPT 平衡
  ↓ Production MD（1-100 ns）
  ↓ 轨迹后处理（去水、RMSD计算、能量分解）
  ↓ MM/GBSA 结合自由能计算
```

### 5.2 对纳米抗体的关键考量

**CDR3 loop 的柔性**：纳米抗体结合靶点的核心是 H3 loop。1 ns 的 MD 长度不足以看到 loop 的大尺度构象变化，但对于评估界面稳定性已经足够。100 ns 的 MD 可以观察完整的结合构象变化。

**二硫键处理**：纳米抗体框架区的经典二硫键（C23-C104）需要在 MD 中正确保持。当前的 `fix_disulfide.py`（CYS→CYX 转换）就是专门处理这个的。

**结合自由能计算**：MM/GBSA 是较为高效的方法，适合在几十个候选之间做相对排序。通过分析能量的标准差来评估候选之间是否有足够的统计区分度。

**选中原因**：MM/GBSA 是当前算力约束下对数十个候选做结合能量排序的最实用方法。如果想更精确但仍可接受更高计算开销，可以改用 MM/PBSA 或 TI/FEP。

### 5.3 多 GPU 并行

支持多 GPU 并行分配（`run_amber_31_driver.py`），最大支持 8 GPU 并行处理不同结构，可大幅缩短 MD 的计算时间。

### 5.4 结果解读

| 指标 | 意义 | 筛选依据 |
|------|------|---------|
| **ΔG_bind** | 结合自由能 | 越负越好，代表结合更强 |
| **ΔG_bind_std** | 能量波动标准差 | 标准差小代表结合模式稳定 |
| **RMSD_bb** | 骨架均方根偏差 | 越小代表结构在 MD 中越稳定 |
| **RMSD_interface** | 界面区骨架偏差 | 直接反映界面是否稳定 |

典型判断：
- ΔG_bind 优于 WT（或优于 baseline）且波动较小的候选优先
- 结构上 CDR3 loop 在 MD 中保持稳定接触的为佳
- RMSD 快速上升或能量持续发散的结构应剔除

## 六、纳米抗体特有考虑

### 6.1 CDR3 长度分布

| CDR3 长度 | 特点 | 筛选调整 |
|-----------|------|---------|
| 3-8 aa | 短 loop，刚性较强 | 适合深 pocket 靶点，可收紧 clash 阈值 |
| 9-15 aa | 典型长度，中等柔性 | 标准流程即可 |
| 16-25 aa | 长 loop，高度柔性 | 需要更长 MD 观察稳定性，pLDDT 可适当放宽 |

### 6.2 框架区稳定性

纳米抗体的框架区（FR1-FR4）具有高度保守的 β-sandwich 结构。如果 AF3 预测的框架区 pLDDT < 0.8，该设计应优先淘汰，因为框架区都不稳定，CDR3 的精确展示就无从谈起。

### 6.3 界面互补性

基于 `calculate_interface_area`（Stage 3 聚类基础）的思路：通过计算界面残基对的距离和接触面积，可以定量评估纳米抗体 CDR3 与抗原表位的互补程度。这部分可以在 Part 3 的轨迹分析中作为补充指标。

## 七、推荐资源分配

| 阶段 | 候选数量 | 计算资源 | 耗时 |
|------|---------|---------|------|
| Part 1 Stage 1 | 数万 → 数千 | CPU 8 核 | 1-2 小时 |
| Part 1 Stage 2+3 | 数千 → 数百 | CPU 8 核 | 1-2 小时 |
| Part 2 PyRosetta | 数百 → 数十 | CPU 8 核 | 0.5-2 小时 |
| Part 3 MD 1ns | 数十 → 10 个 | 4-8 GPU | 2-4 小时 |
| Part 3 MD 100ns | 10 个 → 5 个 | 8 GPU | 2-3 天 |

**最佳实践**：先做 1ns 快速筛选 Top-5，再对这 5 个做 100ns 正式模拟。避免在全部候选上跑长 MD。

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| AF3 预测质量差（全部 pLDDT < 0.7） | 检查输入序列是否正确、靶点结构是否合理，考虑用 ColabFold 或 Chai-1 交叉验证 |
| 筛选过于严格，0 个通过 | 回退阈值，先看指标分布，再分步调整 |
| MM/GBSA 结果与实验不符 | 确认 MD 是否充分平衡、溶剂模型是否合适，可改 MM/PBSA 对比 |
| 计算资源不够 | 缩减产时间（1ns 足以识别明显不稳定的结构），采用多轮筛选策略 |
| CDR3 loop 在 MD 中过度运动但实际结合稳定 | 这部分需靠实验验证，计算可提供"结合模式稳定性区间"，但无法替代实验数据 |

## 九、整体设计路线图

```
Weeks 1-2: 靶点分析 + 序列设计
  → 确定纳米抗体文库构建策略（CDR3长度选择、随机化策略）
  
Weeks 3-4: AF3 批量预测
  → 使用 AlphaFold3 服务器或本地部署
  
Weeks 5-6: Part 1 筛选 + 聚类
  → 三阶段分析，确定 Top 结合模式
  
Week 7: Part 2 PyRosetta 分析
  → 快速物理评分，聚焦 Top-10
  
Weeks 8-9: Part 3 MD + MM/GBSA
  → 1ns 快速筛选 → Top-5 做 100ns
  
Week 10: 最终选择 3-5 个候选
  → 输出综合评分报告 → 进入实验验证
```
