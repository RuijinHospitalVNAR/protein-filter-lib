# 平衡模式：适合 AF3 预测结构

## ✅ 为什么平衡模式更适合 AF3 预测结构？

### AF3 预测结构的特点

1. **整体质量较好**：
   - AF3 学习的是统计分布，预测的整体折叠通常准确
   - 但可能存在局部物理不合理性（键长、键角、侧链堆积）

2. **需要保持预测的折叠**：
   - AF3 预测的折叠通常是合理的
   - 不应该大幅改变整体结构

3. **需要修正物理不合理性**：
   - 局部的不合理性需要修正
   - 使结构符合物理力场

---

## 🎯 平衡模式配置

### 当前配置（平衡模式）

```python
fr.max_iter(2)                                    # 2次迭代（比1次更彻底）
fr.constrain_relax_to_start_coords(True)          # ✅ 约束到起始坐标（保持AF3折叠）
mm.set_jump(False)                                 # ✅ 禁止整体链移动（保持相对位置）
# min_type 使用默认（更稳定，适合批量处理）
```

### 参数说明

| 参数 | 设置 | 原因 |
|------|------|------|
| **max_iter** | 2 | 比1次更彻底，但不会太慢 |
| **constrain_coords** | True | 保持AF3预测的折叠，避免大幅改变 |
| **set_jump** | False | 保持链之间的相对位置（保持结合模式） |
| **min_type** | 默认 | 更稳定，适合批量处理 |

## 📊 与快速模式的对比

### 快速模式（之前）

```python
max_iter = 1
constrain_coords = False
set_jump = True（默认）
```

**特点**：
- ✅ 更快（30-60秒/结构）
- ⚠️ 可能改变结构较大
- ⚠️ 可能偏离AF3预测

### 平衡模式（当前）

```python
max_iter = 2
constrain_coords = True
set_jump = False
```

**特点**：
- ✅ 更彻底（60-120秒/结构）
- ✅ 保持AF3预测的折叠
- ✅ 保持相对位置
- ✅ 结果更准确

## 💡 为什么 min_type 用默认？

### 默认算法的优势

1. **稳定性**：
   - 简单梯度下降，稳定可靠
   - 不容易出现收敛问题

2. **适合批量处理**：
   - 内存占用低
   - 适合大规模并行

3. **足够有效**：
   - 对于AF3预测结构，默认算法通常足够
   - LBFGS的优势在大规模筛选时不明显

### LBFGS 的适用场景

- **单个结构精细优化**：LBFGS更快
- **大规模批量处理**：默认算法更稳定
- **内存受限**：默认算法更节省

## 🎯 最终配置总结

### 当前配置（平衡模式）

```bash
MAX_ITER=2                    # 2次迭代
RELAX=true                    # 启用Relax
FIXBB=false                   # 允许骨架和侧链移动
```

**脚本内部自动设置**：
- `constrain_relax_to_start_coords(True)` - 保持AF3折叠
- `set_jump(False)` - 保持相对位置
- `min_type` - 使用默认（稳定）

### 适用场景

✅ **适合**：
- AF3 预测结构
- 需要保持预测的折叠
- 需要准确的界面能量评估

❌ **不适合**：
- 对接结构（可能需要更大优化）
- 需要快速初步筛选（可以使用快速模式）
