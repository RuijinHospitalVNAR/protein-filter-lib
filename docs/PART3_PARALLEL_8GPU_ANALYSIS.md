# Part 3 8 GPU 并行运行可行性分析

## 1. 资源需求分析

### CPU 需求

**当前配置（--ntomp 12）**：
- 每个 MD 任务：12 个 OpenMP 线程
- 8 个并行任务：12 × 8 = **96 线程**
- **风险**：可能 CPU 过载（取决于系统核心数）

**优化配置（--ntomp 6，推荐）**：
- 每个 MD 任务：6 个 OpenMP 线程
- 8 个并行任务：6 × 8 = **48 线程**
- **风险**：低（大多数系统有 64+ 核心）

**保守配置（--ntomp 4）**：
- 每个 MD 任务：4 个 OpenMP 线程
- 8 个并行任务：4 × 8 = **32 线程**
- **风险**：很低

**性能影响**：
- GPU 加速的 MD 中，CPU 主要用于数据交换和 I/O
- 降低 OpenMP 线程数影响 < 5%
- **建议使用 --ntomp 6**

### 内存需求

**每个任务**：
- MD 运行时：~500 MB
- MM/PBSA 计算：~2 GB
- 中间文件：~1 GB
- **总计**：~3.5 GB/任务

**8 个并行任务**：
- **总需求**：~28 GB
- **系统可用**：~464 GB 空闲（503 GB 总内存）
- **结论**：✅ **内存充足**

### GPU 需求

- **每个任务**：1 GPU
- **8 个并行任务**：8 GPU
- **系统可用**：8 × RTX 4090
- **结论**：✅ **GPU 充足**

## 2. CPU 过载风险评估

### 风险分析

**高负载场景（--ntomp 12）**：
- 8 任务 × 12 线程 = 96 线程
- **风险**：中等（取决于系统 CPU 核心数）

**优化场景（--ntomp 6，推荐）**：
- 8 任务 × 6 线程 = 48 线程
- **风险**：低

**保守场景（--ntomp 4）**：
- 8 任务 × 4 线程 = 32 线程
- **风险**：很低

### 建议配置

```bash
# 推荐：--ntomp 6
--ntomp 6  # 8任务 × 6 = 48线程，安全且高效

# 保守：--ntomp 4
--ntomp 4  # 8任务 × 4 = 32线程，非常安全
```

## 3. 30 个结构时间估算

### 配置：1 ns NPT + 10 ns Production

**单个结构时间**：
- 预处理：2 分钟
- MD（1 ns NPT + 10 ns Production）：约 24 分钟
  - NPT (1 ns)：2 分钟
  - Production (10 ns)：22 分钟（10 × 2.2 分钟/ns）
- MM/PBSA：10 分钟
- **总计**：约 **36 分钟/结构**

### 串行运行（1 个 GPU）

- **总时间**：30 × 36 = **1080 分钟 = 18 小时**

### 并行运行（8 个 GPU）

- **批次数**：⌈30 ÷ 8⌉ = 4 批
- **每批时间**：36 分钟（8 个结构同时运行）
- **总时间**：4 × 36 = **144 分钟 = 2.4 小时**

### 时间对比

| 方案 | 总时间 | 加速比 |
|------|--------|--------|
| 串行（1 GPU） | 18 小时 | 1x |
| 并行（8 GPU） | 2.4 小时 | **7.5x** |
| **节省时间** | **15.6 小时** | - |

## 4. 可行性结论

### ✅ 8 GPU 并行运行**完全可行**

**资源充足**：
- ✅ GPU：8 个可用，满足需求
- ✅ 内存：~28 GB 需求，系统有 ~464 GB 空闲
- ✅ CPU：使用 `--ntomp 6` 时，48 线程需求，大多数系统可满足

**性能优势**：
- ✅ **7.5 倍加速**（30 个结构：18 小时 → 2.4 小时）
- ✅ 充分利用硬件资源

**稳定性**：
- ✅ 降低 `--ntomp` 到 6 或 4，避免 CPU 过载
- ✅ 每个任务独立目录，互不干扰
- ✅ 可监控每个 GPU 的进度

## 5. 使用建议

### 推荐配置

**推荐主入口**：使用配置驱动的 `scripts/run_part3.py`（一条命令跑全流程）：

```bash
# 推荐：配置驱动，8 GPU 并行
python3 scripts/run_part3.py --config config/part3_100ns.yaml --n_gpu 8
```

**历史脚本**：`run_part3_8gpu_100ns.sh` 已由 `scripts/run_part3.py` 替代；历史实现见 `archive/legacy_part3/` 与 [CLEANUP_SUMMARY.md](CLEANUP_SUMMARY.md)。

### 监控建议

```bash
# 监控所有 GPU 使用情况
watch -n 5 nvidia-smi

# 监控 CPU 使用
htop

# 检查各任务进度
tail -f $PART3_OUTPUT_BASE/gpu*/run*.log
```

### 注意事项

1. **CPU 线程数**：建议使用 `--ntomp 6` 或 `4`，避免过载
2. **磁盘空间**：每个结构约 2-3 GB，30 个结构约 60-90 GB
3. **网络 I/O**：如果输入文件在网络存储，可能成为瓶颈
4. **错误处理**：某个任务失败不影响其他任务

## 6. 总结

### ✅ 8 GPU 并行运行**完全可行且推荐**

- **资源充足**：GPU、内存、CPU 都满足需求
- **性能优异**：7.5 倍加速，30 个结构从 18 小时降至 2.4 小时
- **稳定性好**：降低 OpenMP 线程数可避免 CPU 过载
- **易于管理**：使用提供的脚本可一键启动

**建议**：使用 `--ntomp 6` 配置，平衡性能和稳定性。
